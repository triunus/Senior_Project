1. Image Component
 - 화면에 이미지를 표시하는 UI 컴포넌트 (Sprite 2D and UI Type)
 - "Button", "Toggle", "Slider", "Scrollbar", "Dropdown", "Inputfield", "Scroll View"와 같이
   화면에 이미지를 표시하는 모든 UI에 "Image" 컴포넌트가 포함되어 있다.

1) Source Image 변수
 - 해당 변수에 등록되는 이미지가 게임 화면에 출력되며, 이미지가 등록되어 있어야 Image Type과 같은 추가 변수가 Inspector View에 뜬다.

1) Image Type 변수
 - Simple, Sliced, Tiled, Filled 등이 존재한다.

(1) Simple
 - Image Component를 갖고 있는 오브젝트의 크기에 따라 이미지의 크기를 동일하게 조정한다.

(2) Sliced
 - 이미지가 크기가 바뀌었을 때, 미리 설정한 모서리(3*3 Sprite Division Model)는 크기가 그대로 유지되고, 중앙 부분만 크기가 늘어나거나 줄어든다.

(3) Tiled
 - 이미지를 바둑판 형태로 배열하여 출력한다.
 - 3*3 Sprite Division Model가 설정되어 있으면 중앙 부분만 바둑판 형태로 출력하고 끝에 모서리를 출력해준다.

(4) Filed
 - 이미지를 원하는 방향, 원하는 형태로 채워서 출력한다.

(4-1) Filed Mode Variables
 - Fill Method : 채워지는 방식 (Horizontal  |  Vertical  |  Radial 90/180/360)
 - Fill Origin : 채워지는 시작 방향 (Left, Right | Top, Bottom | Bottom Left, Top Left, Bottom Right, Top Right(오른쪽 위 모서리)  | BTLR | BTLR )
 - Fill Amount : 채워진 양
 - Clockwise : 채워지는 방향(시계 방향, 반 시계 방향) <- Fill Method가 Radial일 때 사용.



2. Mask
 - "Mask" 컴포넌트를 가지고 있는 Image UI의,
   자식으로 배치되는 UI들을 부모 오브젝트의 모양에 한정해서 보여주는 것.

 - 카카오톡 프로필에서, 사진의 동그란 부분만 보여주는 것과 같은 기능
 - https://www.youtube.com/watch?v=QnJp45U_UEs, 10분 50초 참고.



3. UI Effects [Outline, Shadow]
 - 처리 능력이 별로 좋지 않아, 사용을 잘 안한다.

1) Outline 변수
 - "Text", "Image"의 외곽선을 표시한다.

2) Shadow 변수
 - "Text", "Image"의 그림자를 표시한다.

 --> 현재 TextMeshPro - Text에서 위의 두 기능을 제공하고 있기에, 사용을 거의 안한다.



4. RawImage Component
 - 게임 화면에 이미지를 표시하는 UI 컴포넌트 (Default Type)
   - 텍스처의 타입이 Sprite (2D and UI)가 아닐 때 사용한다.
   - (랜더 텍스처 or 특수한 텍스처를 사용할 때 사용)

 - 이미지를 표현하는 대부분의 UI는 "Image" 컴포넌트를 사용한다.


※ Texture Type - Sprite (2D and UI)
 - Asset으로 사용되는 Image들의 Inspector View를 보면, Texture Type 값이 존재한다.
 - Texture Type이 Sprite(2D and UI) 값을 가져야, 해당 Asset을 Image Componet에 등록할 수 있다.

※ 3*3 Sprite Division Model
 - Asset으로 사용되는 Image들의 Inspector View를 보면, Sprite Editor 버튼이 보인다.
 - 해당 image의 Sprite Editor의 Border값을 조정하여, 적용 부분을 조정할 수 있다.
